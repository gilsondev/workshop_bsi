Modificadores de Acesso

Modificadores de acesso

São definições de visibilidade a ser usado nas classes, atributos e métodos

  • default
  • public
  • private
  • final
  • static
  • protected
  • abstract

Atenção!

  1. Classes não usam o modificador private
  2. Atributos e métodos não são visíveis pelas classes herdadas

Herança

Criando classes a partir de outras classes

Herança

Esse princípio serve para reutilização de código, sendo que uma classe herda os comportamentos e atributos de outra classe.

  • Nova classe: Subclasse (Classe filha)
  • Classe antiga: Superclasse (Classe pai)

Polimorfismo

Usando elementos de formas diferentes

Polimorfismo

Pertencente da Orientação a Objetos, permite usar um elemento de formas diferentes

Existem dois tipos:

  1. Sobrescrita
  2. Sobrecarga

Observações sobre a Sobrecarga

Em métodos com a mesma assinatura, mas com pequenas diferenças entre elas, fica a cargo do compilador escolher qual método será invocado de acordo com os argumentos escolhidos.

Quando você define um construtor para sua classe, você estå fazendo uma sobrecarga.

Classes abstratas

Modelo para classes concretas

Classes abstratas

Serve como um "template" para classes que forem seguir sua estrutura.

Para definir uma classe abstrata, é usado o modificador abstract

Características de uma classe abstrata

  • Não pode ser instanciada
  • Sempre devem ser herdadas por classes concretas
  • Podem conter métodos com ou sem implementação (métodos abstratos)
